# Day 01 - Shell basic and file system in Linux

## Objective

Get comfortable in the Linux terminal
 To learn basic command and understanding the filesystem structure in linux.

---

## What I Learned
A shell is a program that allows a user to interract with a linux system via commands. it acts as a bridge between the user and the operating system. Common shell includes, bash, Zsh, sh etc.
I also learnt and interracted with the shell using the listed command
- mkdir: creates a new directory (folder)
- cd: change directory
- pwd shows my exact location in the filesystem
- ls -la shows all files including hidden ones with permissions
- ~ always means my home directory (/home/user19)
- > creates/overwrites a file,
- >> appends to a file
- The server runs UTC time — important for pipeline scheduling

  Away from the shell, I also learnt how to clone a github and push to a repo in my gitHub.

## What I Built / Practiced
- Navigated the filesystem using pwd, cd, ls
- Created orientation.log that captures user, hostname, date, shell

## Challenges Faced
- No sudo access on shared VPS

## Key Takeaways
- Every path on Linux starts from / (root)
- Bash is my shell — everything I automate will be written in it
- Always log metadata (date, user, host) in data scripts
